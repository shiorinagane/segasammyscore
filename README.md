# スコア速報ランキング LIFF（スプレッドシート駆動型）

セガサミー チャリティープロアマ大会の「総合ランキング」をLINE内（LIFF）で表示する仕組み。

- ランキングのデータは **Googleスプレッドシートの配信シート**（配信1〜配信4）で管理
- 各シートのB1に何か入力（チェックボックスのONでも「OK」「公開」などの文字でもよい）したシートがページに反映される
- **LINEで配るURLは4回とも同じ**（LIFF URL 1本）。ページを開くたびに公開中のシートを読みに行くため、URLの差し替えは不要

## 全体の流れ

```
LINE配信（4回：前半速報 / ハーフ集計 / 後半速報 / 最終結果）
  └─ LIFF URL（https://liff.line.me/xxxx）← 毎回同じURL
       └─ index.html（GitHub Pages）がLINE内で開く
            └─ GAS doGet ← 「公開」チェックONの配信シートを読んでJSONを返す
```

## 構成ファイル

| ファイル | 役割 |
|---|---|
| `index.html` | ランキング表示ページ（GitHub Pagesに置く） |
| `gas_code.gs` | スプレッドシートを読むAPI（GASウェブアプリ） |
| `配信1_前半速報.csv` 〜 `配信4_最終結果.csv` | スプレッドシートに各配信用シートとしてインポートする雛形（サンプルデータ入り） |

## スプレッドシートの作り方

1ファイルに配信シートを4枚作る。シート名の例：

- `配信1_前半速報`
- `配信2_ハーフ集計`
- `配信3_後半速報`
- `配信4_最終結果`

### シートのレイアウト（固定位置）

| | A | B | C |
|---|---|---|---|
| 1 | 公開 | OK（空欄以外なら何でもよい） | |
| 2 | 見出し | 前半スコア速報 | |
| 3 | 説明文 | 10:00時点のチーム戦最新順位をお届けします。 | |
| 4 | 更新時刻 | 7/23 10:40 | |
| 5 | 表示数 | 10（空欄なら全組表示） | |
| 6 | 順位 | チーム名 | ポイント |
| 7 | | 香妻陣一朗チーム | +24 |
| 8 | | 杉本エリックチーム | +2 |
| … | | | |

### ルール

- **B1（公開フラグ）**：空欄でなければ何でもON扱い。「OK」「公開」「ON」のような文字を入れるだけでよい（チェックボックスを使いたい場合はセルB1を選択→「挿入→チェックボックス」でもOK、TRUEもON扱い）。空欄に戻す・文字を消すとOFF
- **見出し（B2）**：ページの「更新 7/23 10:40 時点／◯◯」の◯◯部分に出る
- **説明文（B3）**：タイトル下の一行に出る。**折り返さない設計なので22文字程度まで**
- **更新時刻（B4）**：文字列で `7/23 10:40` と入れるのが確実（日時型でも自動で `7/23 10:40` 形式に変換される）
- **順位（A列）**：**この大会は同点が発生するため、毎回手入力する運用にする**（空欄でも上から自動で1, 2, 3…と振られる仕組みはあるが、同点を考慮しないので事故のもと）。同点は同じ順位、次の順位はタイの人数分スキップする（例：4位が2チーム同点なら両方 `4`、次は `6`）
- **チーム名（B列）**：空欄の行で読み込み終了。**上位から順に並べて入力する**
- **ポイント（C列)**：`+24` `-3` `0` など。`+`付き文字列でも数値でもOK（全角の−も自動変換）
- **表示数（B5）**：ページに何位まで出すかを数値で指定。**データ自体は毎回全組入力してOK**（表示数は別枠で絞れる）
  - 前半速報・後半速報など**トップ10だけ見せたい配信は `10` と入力**（サンプルCSVは配信1・配信3に `10` を設定済み）
  - ハーフ集計・最終結果など**全組見せたい配信は空欄のまま**（サンプルCSVは配信2・配信4が空欄）

### 複数シートのB1が同時に入力されているときは？

一番右（＝番号の大きい配信シート）が優先される。とはいえ事故防止のため、**新しい配信を公開したら前のシートのB1は空欄に戻す**運用を推奨。

## 初回セットアップ（1回だけ）

### 1. スプレッドシート

1. 新規スプレッドシートを作成（例：「セガサミープロアマ_スコア速報」）
2. 次の4本のCSVを、それぞれ別シートとしてインポート（見出し・説明文はPDF「20260709_プロアマ予想ダービー.pdf」の配信文に合わせた仮テキスト。当日の実際の時刻・文言に書き換えて使う）
   - `配信1_前半速報.csv`
   - `配信2_ハーフ集計.csv`
   - `配信3_後半速報.csv`
   - `配信4_最終結果.csv`
   - インポート方法：シート下部の「＋」でシートを追加 →「ファイル → インポート → アップロード」→ 該当CSVを選択 →「新しいシートを挿入」を選択
3. 4枚それぞれのB1に「OK」など空欄以外の何かを入力（チェックボックスにしたい場合はセルB1を選択→「挿入→チェックボックス」でもよい）
4. データはサンプル（1組〜34組・34組フル。運営マニュアル記載の34組に合わせている。ポイントは仮の数字で自然な散らばりを持たせ、同点も3箇所ほど混ぜてある）なので、当日はA列の順位・B列のチーム名・C列のポイントを実際の値に差し替える。**同点があるためA列は毎回入力する**（同点は同じ数字、次はタイの人数分スキップ）
5. なお運営マニュアル（`20260706_SSPA2026_運営マニュアル0702準備①稿.pdf` p17）によると、チーム戦のポイントはプロのボギー=-1・ダブルボギー以上=-3（DOOMホール＝6H/14Hで男子プロは3倍→最大-9）、アマはダブルボギー以上=-1と、**マイナスも仕様として起こり得る**。ページ上ではポイントに＋は付けず数値のみで表示するが、マイナスは入力どおり（例：-8→「-8」）そのまま表示する運用に決定。0未満のチームもマイナス表記で正しく区別できる
6. URLの `/d/〇〇〇/` 部分（スプレッドシートID）を控える

### 2. GAS

1. スプレッドシートのメニュー「拡張機能 → Apps Script」を開く
2. `gas_code.gs` の中身を貼り付け、`SPREADSHEET_ID` を書き換える
3. 「デプロイ → 新しいデプロイ → ウェブアプリ」
   - 実行ユーザー：**自分**
   - アクセスできるユーザー：**全員**（LINEユーザーが匿名で叩くため必須）
4. 発行された `https://script.google.com/macros/s/…/exec` のURLを控える

コード修正時は「デプロイを管理 → 編集 → 新バージョン」で更新するとURLが変わらない。「新しいデプロイ」を作るとURLが変わるので注意。

### 3. GitHub Pages に index.html を置く

1. `index.html` 冒頭の `GAS_URL` を手順2のURLに書き換える（`LIFF_ID` は手順4で確定するので仮のままでよい。`SITE_URL` に大会公式サイトURLを入れるとフッターにリンクが出る）
2. GitHubにリポジトリを作成（private不可。**public**にする。ページ内に個人情報は出ないので問題なし）

```bash
cd score_ranking   # index.htmlのあるフォルダ
git init
git add index.html
git commit -m "add score ranking page"
git branch -M main
git remote add origin https://github.com/<アカウント名>/<リポジトリ名>.git
git push -u origin main
```

3. GitHubのリポジトリ画面 → Settings → Pages → Branch を `main` / `(root)` にして Save
4. 数分後に `https://<アカウント名>.github.io/<リポジトリ名>/` で公開される。ブラウザで開いて表示確認（この時点でランキングが出ればGAS連携OK）

### 4. LINE Developers で LIFF アプリを作成（LINEログインチャネル）

LINEログインチャネルで作成する方針で確定（ミニアプリ限定の要件・審査を気にする必要はない）。

1. https://developers.line.biz/ に、セガサミー公式アカウントを開設したLINEビジネスIDでログイン
2. セガサミー公式アカウントのMessaging APIチャネル（パーティーコネクト連携で使用中のもの）が入っている**同じプロバイダー**を開く
   - **別プロバイダーに作らないこと**。同じ人でも別のユーザーIDが振られるため、後からクイズ回答者と「誰が開いたか」の名寄せができなくなる
3. プロバイダー内「新規チャネル作成」→「**LINEログイン**」
   - チャネル名：「セガサミープロアマ ランキング」など（認証画面に表示されることがあるので内部用の名前は避ける）
   - アプリタイプ：**ウェブアプリ**
4. 作成したチャネル内「LIFF」タブ → 追加
   - LIFFアプリ名：スコア速報ランキング（LINE内ヘッダーに出る名前）
   - サイズ：**Full**（ランキングは全画面表示が見やすい）
   - エンドポイントURL：手順3のGitHub Pages URL（https必須・実際に開けるURLを末尾まで正確に）
   - Scope：`profile` のチェックだけでOK
   - ボットリンク機能：Off（開く人は配信経由＝すでに友だちのため不要）
5. 発行された LIFF ID を `index.html` の `LIFF_ID` に反映して再push（数分でPagesに反映。確認はシークレットタブで）
6. **チャネルのステータスを「開発中」→「公開済み」に切り替える**
   - 開発中のままだと管理者・テスター以外はLIFFを開けない。「自分のスマホでは開けるのに他の人は開けない」の典型原因
7. 完成した配信用URL：`https://liff.line.me/<LIFF ID>`（4回の配信すべてこのURL）

補足：GitHub Pages側のURLを後から変えた場合は、LIFF設定のエンドポイントURLを直せばよい。LIFF URL自体は変わらない。

## 当日の運用（配信のたびにやること）

1. ブリヂストンスポーツから共有された順位データを、該当の配信シートに入力（チーム名を上位から順に、ポイントも）
2. B4の更新時刻を書き換える（例：`7/23 10:40`）
3. 入力が終わったら **B1に「OK」など何か入力して公開状態にする**
4. 前の配信シートのB1は空欄に戻す（OFFにする）
5. LINE公式アカウントから配信文＋いつものLIFF URL（`https://liff.line.me/<LIFF ID>`）を配信

シートの入力が終わる前にB1へ入力しないこと（入力途中の状態が公開される）。

## 動作確認のチェックリスト

- [ ] ブラウザでGitHub Pages URLを開くとランキングが表示される
- [ ] LINEアプリでLIFF URLを開くとランキングが表示される
- [ ] **チャネル管理者ではない人のスマホ**でもLIFF URLが開ける（チャネルの「公開済み」切り替え漏れの検出）
- [ ] B1をすべて空欄にすると「ランキングは準備中です」と表示される
- [ ] 配信1のB1に入力→配信2のB1に入力に切り替えると、ページ再訪で配信2の内容に変わる
- [ ] ポイントは＋なしの数値で表示され、マイナスは「-1」のように反映される（色分けはしない）
- [ ] 表示数（B5）に `10` と入れたシートは、34組分データを入れていても10位までの表示になる
- [ ] 表示数（B5）を空欄にしたシートは、全組（34組）表示になる
- [ ] チーム同点で同順位を手入力したとき、その順位のまま表示される
